## 🚀 1. 로컬 실행 (가장 간단)

```bash
# Python 3.9+ 권장
pip install -r requirements.txt
streamlit run app.py
```

브라우저에서 자동으로 `http://localhost:8501` 이 열립니다.

> **솔버 주의** — 본 앱은 PuLP에 내장된 **CBC 솔버**를 사용하므로 별도 설치(GLPK 등)가 필요 없습니다. 강의록의 Pyomo+GLPK 코드와 수치 결과는 동일합니다 (위 검증표 참조).

## ☁️ 2. Streamlit Cloud 무료 배포 (5분)

1. **GitHub 저장소 생성** — `app.py`, `requirements.txt`, `.streamlit/config.toml` 을 모두 커밋·푸시
2. [share.streamlit.io](https://share.streamlit.io/) 접속 → GitHub 로그인
3. **New app** → 저장소·브랜치·`app.py` 선택 → **Deploy** 클릭
4. 1~2분 후 `https://<앱이름>.streamlit.app` 형태의 공개 URL 발급

> 💡 push 만 해두면 **수요·비용·전략이 바뀐 새 버전도 자동 재배포** 됩니다.

### 다른 배포 옵션
- **Render / Railway / Fly.io** — `Procfile`을 한 줄 추가하면 됩니다:
  `web: streamlit run app.py --server.port=$PORT --server.address=0.0.0.0`
- **Hugging Face Spaces (Streamlit 템플릿)** — `requirements.txt`만 있으면 동일하게 작동

## 🧭 3. 사용 흐름

### (1) 좌측 사이드바에서 입력
- **계획 기간 · 수요** : 슬라이더로 개월 수 변경 → 표에서 월별 수요 직접 편집
- **초기 / 종료 조건** : 초기 인력·재고, 최종 재고 목표
- **생산 능력** : 작업일·작업시간·표준시간·초과근무 한도
- **비용 (천원)** : 8가지 비용 항목 (정규/초과/고용/해고/재고/부재고/재료/하청)
- **전략** : `LP 최적화` / `평준화` / `추종` 택1, IP 모드 토글

### (2) 우측 대시보드 자동 갱신
- 5개 KPI 카드 (총비용, 추정이익, 수요/공급, 평균인력, 총 초과근무)
- 자동 진단 메시지 (부재고/외주/초과근무 강도)
- 핵심 차트 4개 + 인력운영 차트
- **전략 비교 패널** — 3가지 전략을 자동으로 모두 풀어 비용·인력·재고 추이를 나란히 표시
- **자동 추천 전략** — 실현가능한 전략 중 최저비용 선택 + 현재 선택과의 차액 표시

### (3) 결과 다운로드
- 시점별 결정변수(W/H/L/P/I/S/C/O) 테이블 → `CSV 다운로드` 버튼

## 🧮 4. 수리적 모델 (요약)

**결정변수**
$W_t, H_t, L_t, P_t, I_t, S_t, C_t, O_t \ge 0$

**목적함수 (단위: 천원)**
$$\min Z = \sum_t \bigl(640 W_t + 6 O_t + 300 H_t + 500 L_t + 2 I_t + 5 S_t + 10 P_t + 30 C_t\bigr)$$

**제약**
- 인력 균형: $W_t = W_{t-1} + H_t - L_t$
- 생산능력: $P_t \le 40 W_t + O_t/4$
- 재고 균형: $I_t = I_{t-1} + P_t + C_t - D_t - S_{t-1} + S_t$
- 초과근무 한도: $O_t \le 10 W_t$
- 초기/최종: $W_0=80, I_0=1000, I_T \ge 500, S_0=S_T=0$

**전략별 추가 제약**
| 전략 | 추가 제약 | 의미 |
|---|---|---|
| LP 최적화 | (없음) | 모든 옵션을 동시에 고려하여 비용 최소화 |
| 평준화 | $H_t = L_t = 0$ | 인력 변화 없이 일정한 작업자로 운영 |
| 추종 | $I_t \le I_T^{target}$ | 재고를 최소화 → 생산이 수요를 추종 |

## 🛠 5. 트러블슈팅

| 증상 | 원인 / 해결 |
|---|---|
| 배포 시 `pulp` 솔버 오류 | `requirements.txt`에 `pulp>=2.7`이 있는지 확인 (CBC가 함께 설치됨) |
| "해를 찾을 수 없습니다" | 평준화에서 인력이 부족 / 추종에서 최종재고 목표가 너무 큼 → 다른 전략으로 전환하거나 파라미터 완화 |
| 한글 폰트 깨짐 | Plotly는 시스템 폰트를 사용하므로 OS에 한글 폰트가 있으면 자동 표시됨 (Streamlit Cloud는 정상) |
| 결과가 약간 다름 | LP/IP 모드와 전략 선택이 같은지 확인. CBC와 GLPK는 동일한 LP에서 같은 최적값을 반환합니다. |
